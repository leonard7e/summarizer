[[providers]]
== Providers

A _provider_ is a backend that `summarizer` uses to send prompts and receive completions.
Providers are selected by the prefix in the model identifier (`provider:model_id`).

=== Google Gemini

The `google` (alias: `gemini`) provider uses the https://ai.google.dev/[Google Gemini API].

==== Setup

1. Create an API key at https://aistudio.google.com/[Google AI Studio].
2. Add the key to your configuration:
+
[source,yaml]
----
providers:
  gemini:
    api_key: "AIza..."
----

==== Model Format

[source]
----
google:<model-id>
----

.Examples
[source,bash]
----
summarizer --model google:gemini-1.5-flash document.txt
summarizer --model google:gemini-1.5-pro  document.txt
summarizer --model google:gemini-2.0-flash document.txt
----

==== Notes

* The context window size is queried automatically from the Gemini API at runtime and used to compute batch sizes.
* Use `summarizer list-models` to see all Gemini models currently available to your key.

=== OpenRouter

The `openrouter` provider gives access to hundreds of models from various vendors through a single API key at https://openrouter.ai[openrouter.ai].

==== Setup

1. Create an account and generate an API key at https://openrouter.ai/keys[openrouter.ai/keys].
2. Add the key to your configuration:
+
[source,yaml]
----
providers:
  openrouter:
    api_key: "sk-or-..."
----

==== Model Format

[source]
----
openrouter:<openrouter-model-id>
----

The model ID after `openrouter:` is the full OpenRouter model identifier, which typically includes the vendor name.

.Examples
[source,bash]
----
summarizer --model openrouter:anthropic/claude-3-haiku   document.txt
summarizer --model openrouter:meta-llama/llama-3-8b-instruct document.txt
summarizer --model openrouter:mistralai/mistral-7b-instruct  document.txt
----

==== Notes

* OpenRouter supports free-tier models; check https://openrouter.ai/models[openrouter.ai/models] for current availability.
* Context window sizes are queried from the OpenRouter models endpoint.

=== Ollama (Local)

The `ollama` provider runs models locally via https://ollama.com[Ollama], with no API key required.

==== Setup

1. Install Ollama: https://ollama.com/download[ollama.com/download]
2. Pull a model:
+
[source,bash]
----
ollama pull llama3
----
3. Optionally configure the host and context window in `config.yaml` (defaults are usually fine):
+
[source,yaml]
----
providers:
  ollama:
    host: "http://localhost:11434"
    num_ctx: 4096
----

==== Model Format

[source]
----
ollama:<model-name>
----

.Examples
[source,bash]
----
summarizer --model ollama:llama3       notes.txt
summarizer --model ollama:mistral      notes.txt
summarizer --model ollama:phi3:mini    notes.txt
----

==== `num_ctx` — Context Window

The `num_ctx` configuration value tells Ollama how many tokens to allocate for the context window.
`summarizer` uses this value to calculate batch sizes.

[IMPORTANT]
====
Increasing `num_ctx` beyond a model's training context length has no benefit and may degrade quality.
Consult the model's Ollama page (`ollama show <model>`) for the recommended value.
====

=== OpenAI-Compatible

The `openai-compatible` provider works with any server that implements the OpenAI Chat Completions API.
Common use cases include local inference servers such as https://github.com/vllm-project/vllm[vLLM], https://lmstudio.ai[LM Studio], or https://github.com/ggml-org/llama.cpp[llama.cpp].

==== Setup

[source,yaml]
----
providers:
  openai-compatible:
    api_key: "sk-..."       # May be an arbitrary string for local servers
    base_url: "http://localhost:8080/v1"
----

==== Model Format

[source]
----
openai-compatible:<model-id>
----

The model ID after `openai-compatible:` must match the model name as the server expects it.

.Examples
[source,bash]
----
summarizer --model openai-compatible:my-local-model document.txt
----

==== Notes

* The context window limit is retrieved from the server's `/v1/models` endpoint.
  If the server does not report this, `summarizer` falls back to a conservative default of 8 192 tokens.
* Set `api_key` to any non-empty string for servers that do not enforce authentication.
