[[configuration]]
== Configuration

=== File Location

The configuration file is stored at the platform-specific user configuration directory:

[cols="1,2", options="header"]
|===
| Platform | Path

| Linux
| `~/.config/summarizer/config.yaml`

| macOS
| `~/Library/Application Support/summarizer/config.yaml`

| Windows
| `%APPDATA%\summarizer\config.yaml`
|===

The file is created automatically with default values when `summarizer` runs for the first time, or when you run `summarizer init`.

[IMPORTANT]
====
On Unix systems the file is saved with permissions `0600` (readable only by the owner) because it may contain API keys.
====

=== Complete Example

[source,yaml]
----
default_model: google:gemini-1.5-flash
max_output_tokens: 4096

providers:
  gemini:
    api_key: "AIza..."

  openrouter:
    api_key: "sk-or-..."

  ollama:
    base_url: "http://localhost:11434"
    num_ctx: 8192

  openai-compatible:
    - name: "my-local-server"
      api_key: "sk-..."
      base_url: "http://localhost:8080/v1"
    - name: "another-provider"
      api_key: "sk-..."
      base_url: "https://api.example.com/v1"
----

=== Field Reference

==== Top-Level Fields

[cols="1,1,1,3", options="header"]
|===
| Field | Type | Default | Description

| `default_model`
| string (optional)
| —
| The model used when `--model` is not specified on the command line. Format: `provider:model_id`.

| `max_output_tokens`
| integer
| `4096`
| Number of tokens reserved for the model's output in each request.
|===

[TIP]
====
The default value of `4096` was chosen to work well with Ollama's default context window (which we set to `8192` by default). This leaves enough room for the input prompt and file contents.

If your model and hardware allow it (e.g., modern LLMs with 32k+ context), you should significantly increase these values – especially `num_ctx` for Ollama – (e.g., to `16384` or `32768`) to process even very long documents without truncation or information loss.
====

[CAUTION]
====
The `max_output_tokens` value is critical. It defines the maximum length of the summary generated in each batch. If this value is too small, the model might be forced to cut off the response, leading to **significant information loss**.
====

==== `providers.gemini`

[cols="1,1,1,3", options="header"]
|===
| Field | Type | Default | Description

| `api_key`
| string
| —
| Google Gemini API key. Obtain one at https://aistudio.google.com/[Google AI Studio].
|===

==== `providers.openrouter`

[cols="1,1,1,3", options="header"]
|===
| Field | Type | Default | Description

| `api_key`
| string
| —
| OpenRouter API key. Obtain one at https://openrouter.ai/[openrouter.ai].
|===

==== `providers.ollama`

[cols="1,1,1,3", options="header"]
|===
| Field | Type | Default | Description

| `base_url`
| string
| `http://localhost:11434`
| The base URL of the Ollama server.

| `num_ctx`
| integer
| `8192`
| Context window size to request from Ollama.
|===

==== `providers.openai-compatible`

This is a list of providers. Each entry requires:

[cols="1,1,1,3", options="header"]
|===
| Field | Type | Default | Description

| `name`
| string
| —
| A unique name for this provider. Used as the prefix in the model ID (e.g., `my-local-server:llama3`).

| `api_key`
| string
| —
| API key for the endpoint.

| `base_url`
| string
| —
| Base URL of the OpenAI-compatible API (e.g. `http://localhost:8080/v1`).
|===

=== Managing the Configuration

Use the built-in subcommands to manage the configuration interactively:

[source,bash]
----
# Full setup wizard
summarizer init

# Change only the default model
summarizer default-model

# Check reachable models
summarizer list-models
----
