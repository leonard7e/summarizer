[[usage]]
== Usage

=== Synopsis

[source]
----
summarizer [OPTIONS] [FILES]...
summarizer <SUBCOMMAND>
----

If no `FILES` are provided and the input is piped, `summarizer` reads from standard input (stdin).

=== Arguments

[cols="1,3", options="header"]
|===
| Argument | Description

| `[FILES]...`
| One or more files to summarize. Passed as positional arguments.
  Supports text files and various media formats (images, audio, video).
|===

=== Options

[cols="1,1,3", options="header"]
|===
| Short | Long | Description

| `-p`
| `--prompt <TEXT>`
| Instruction text to prepend to the summarization request.
  If omitted, a default prompt is used: _"Please summarize the following text comprehensively."_

| `-f`
| `--prompt-file <PATH>`
| Path to a file containing the instruction text.
  If both `--prompt` and `--prompt-file` are given, the file content comes first, followed by the inline prompt.

| `-m`
| `--model <PROVIDER:MODEL>`
| Model to use, in the format `provider:model_id` (e.g. `google:gemini-1.5-flash`).
  Overrides the `default_model` from the configuration file.

|
| `--debug`
| Print debug information to standard error: provider, model, context window size, batch sizes, and per-file progress.

| `-h`
| `--help`
| Print help information.

| `-V`
| `--version`
| Print version information.
|===

=== Subcommands

[cols="1,3", options="header"]
|===
| Subcommand | Description

| `init`
| Run the interactive setup wizard. Collects API keys and lets you choose a default model. Creates or overwrites `~/.config/summarizer/config.yaml`.

| `list-models`
| Query all configured providers and print the available model IDs, prefixed with `provider:`.

| `default-model`
| Interactively select a new default model from the list of all available models and save it to the configuration file.
|===

=== Supported Media Types

`summarizer` supports more than just text. You can pass the following file types:

* **Images**: `png`, `jpg`, `jpeg`, `webp`, `gif`
* **Audio**: `mp3`, `wav`, `flac`, `ogg`, `m4a`, `aac`
* **Video**: `mp4`, `mov`, `webm`, `avi`, `mpeg`, `mpg`

[NOTE]
====
Ensure the selected model supports the respective media type. Gemini models typically have broad multimodal support.
====

=== Examples

.Summarize from stdin
[source,bash]
----
cat report.txt | summarizer
----

.Summarize an image
[source,bash]
----
summarizer screenshot.png -p "Describe this image in detail."
----

.Summarize a video
[source,bash]
----
summarizer presentation.mp4 -p "What are the main points discussed?"
----

.Summarize multiple files with a custom prompt
[source,bash]
----
summarizer -p "List all decisions made." minutes_*.txt
----

.Debug mode to inspect batching
[source,bash]
----
summarizer --debug document.pdf.txt
----
