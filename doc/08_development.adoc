[[development]]
== Development

=== Module Overview

[cols="1,3", options="header"]
|===
| Module | Responsibility

| `main`
| Entry point. Parses the CLI, routes subcommands, loads config, and calls into `engine` or `setup`.

| `cli`
| Defines the `Cli` struct and `Commands` enum using https://docs.rs/clap/latest/clap/[clap derive].

| `config`
| Defines the `Config` struct (and nested provider configs). Handles loading from and saving to `config.yaml` via `serde_yaml`.

| `engine`
| Core summarization logic: context-budget calculation, file batching, prompt construction, and the batch processing loop.

| `file`
| Async file reading (`tokio::fs`). Produces `ProcessedFile` values consumed by `engine`.

| `provider`
| Defines the `LlmProvider` trait and the `create_provider` factory function. Contains sub-modules for each provider implementation.

| `provider::gemini`
| Google Gemini API implementation.

| `provider::openai_compatible`
| OpenAI-compatible API implementation (used by both OpenRouter and the `openai-compatible` provider).

| `provider::ollama`
| Ollama HTTP API implementation.

| `setup`
| Interactive initialization wizard and model-listing helpers (uses https://docs.rs/dialoguer/[dialoguer]).
|===

=== Running Tests

[source,bash]
----
cargo test
----

Tests are co-located with the source in `#[cfg(test)]` modules.
The test suite uses https://docs.rs/mockito/[mockito] for HTTP mocking and https://docs.rs/tempfile/[tempfile] for temporary files.

Run only a specific test module:

[source,bash]
----
cargo test config::tests
cargo test provider::tests
----

Run with output from passing tests visible:

[source,bash]
----
cargo test -- --nocapture
----

=== Adding a New Provider

. **Implement `LlmProvider`** â€” create a new file in `src/provider/`, e.g. `src/provider/myprovider.rs`, and implement the trait:
+
[source,rust]
----
use async_trait::async_trait;
use anyhow::Result;
use crate::provider::LlmProvider;

pub struct MyProvider { /* fields */ }

#[async_trait]
impl LlmProvider for MyProvider {
    async fn complete(&self, prompt: &str, model: &str) -> Result<String> {
        todo!()
    }

    async fn list_models(&self) -> Result<Vec<String>> {
        todo!()
    }

    async fn get_context_limit(&self, model: &str) -> Result<usize> {
        todo!()
    }
}
----

. **Register the module** â€” add `pub mod myprovider;` to `src/provider/mod.rs`.

. **Add a config struct** â€” if your provider needs configuration (API keys, URLs), add a struct to `src/config.rs` and include it in `ProvidersConfig`.

. **Register in the factory** â€” add a match arm for your provider name in `create_provider` inside `src/provider/mod.rs`:
+
[source,rust]
----
"myprovider" => {
    let conf = config.providers.myprovider
        .as_ref()
        .ok_or_else(|| anyhow!("MyProvider not configured"))?;
    Ok(Box::new(myprovider::MyProvider::new(conf.api_key.clone())))
}
----

=== Coding Conventions

* **Rust edition**: 2024 (set in `Cargo.toml`).
* **Error handling**: Use `anyhow::Result` and `anyhow::anyhow!` throughout. Do not use `unwrap()` outside of tests.
* **Async**: The binary uses `tokio` with the `full` feature set. Provider methods are `async` via `async-trait`.
* **Serialization**: `serde` + `serde_yaml` for config; `serde_json` for API request/response bodies.
* **No `println!` to stderr**: Progress and debug information goes to `eprintln!`; only the final summary goes to `println!` (stdout), so output can be redirected cleanly.

=== Building the Documentation

From the project root:

[source,bash]
----
# Single-page HTML
asciidoctor doc/summarizer.adoc -D doc/out/

# PDF
asciidoctor-pdf doc/summarizer.adoc -D doc/out/
----

Output files are written to `doc/out/` (created automatically).
The `doc/out/` directory is not tracked by Git.
